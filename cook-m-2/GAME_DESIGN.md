## 完整四阶段流程

### 阶段一：准备材料

| 操作 | 交互 | 结果 |
|------|------|------|
| 面饼 | 点击 | 飞入铁板 |
| 鸡蛋 | 点击 🐔**母鸡** | 蛋飞入**食材盒** |
| 洋葱 | 点击 [🧅洋葱块] **3次切碎** | 碎洋葱飞入**食材盒** |
| 烤肠 | 拖入铁板 → Timer烤制 → 点击烤好的烤肠 | 直接飞到翻好面的面饼上 |
| 辣酱 | 点击 [🌶辣酱桶] | 辣酱飞入**食材盒** |
| 纸盒 | 点击 [📦纸盒堆] | 在铁板上方放置空纸盒（最多3个） |

### 阶段二：铁板制作

| 步骤 | 操作 | 说明 |
|------|------|------|
| 放面饼 | 点击 [面饼] | dish 为空时可用 |
| 打蛋 | 点击**食材盒**[🥚] | 有面饼时可用 |
| 摊蛋 | 自动 Timer(2s) | 自动 |
| 翻面 | **向上滑动**铁板 | 蛋已摊 |
| 加烤肠 | 点击烤好的烤肠 | 已翻面，自动飞到面饼上 |
| 加洋葱 | 点击**食材盒**[🧅] **最多3次** | 每次增加填充度，影响价格 |
| 加辣酱 | 点击**食材盒**[🌶] **最多3次** | 每次增加填充度，影响价格 |
| 卷起 | **左右滑动**铁板 | 已翻面+有料 |
| 切段 | **点击铁板 x5** | 已卷起 |

### 阶段三：装盒
- **拖拽**切好的冷面 → 放入铁板上方的空纸盒

### 阶段四：出餐
- **拖拽**装了面的纸盒 → 最前面顾客 → 订单匹配：
  - ✅ 匹配 → 顾客开心离开 + 按价值加钱
  - ⚠️ 不匹配 → 按**价值减半**加钱（面没白做，但客人讨价还价）

## 每日倒计时

- 每天限时 **3 分钟**
- HUD 顶部显示倒计时（MM:SS）
- 倒计时归零 → **本日强制结束**，不等待当前订单完成
- 铁板上的半成品、食材盒中的材料全部清空，直接进入日结
- 顾客生成速度随天数加快，倒计时不变

## 填充度价值计算

```gdscript
func calculate_value(base_price: int) -> int:
    var fill_ratio = 0.0
    fill_ratio += onion_fill / 3.0 * 0.5
    fill_ratio += sauce_fill / 3.0 * 0.5
    var value = int(base_price * (0.3 + fill_ratio * 0.7))
    return max(1, value)
```

- onion_fill=0, sauce_fill=0 → 30% 价值
- onion_fill=3, sauce_fill=3 → 100% 价值
- onion_fill=2, sauce_fill=1 → 约 65%

**订单不匹配时：** 最终收入 = `calculate_value() / 2`

## 交互方式

| 交互 | 适用场景 |
|------|---------|
| 点击→飞入 | 食材/酱料点击，Tween飞到目标 |
| 多次点击 | 切洋葱(3次)、填洋葱/辣酱(最多3次) |
| 拖拽(到区域) | 烤肠拖入铁板、冷面拖入纸盒 |
| 拖拽(到顾客) | 装面纸盒拖到最前面顾客 |
| 向上滑动(铁板) | 翻面 |
| 左右滑动(铁板) | 卷起 |
| 点击(铁板) | 切段(累计5次) |
